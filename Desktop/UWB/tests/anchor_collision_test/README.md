# 테스트 A — 다중 앵커 충돌 곡선 측정

> 단일 RF 채널에서 앵커 **1대 vs 2대**가 동시에 폴링할 때 no-range 발생률(PER)이
> 얼마나 오르는지 측정한다. **100대 확장이 물리적으로 가능한지 판단하는 1차 데이터.**

이전 분석에서 식별한 **1순위 병목(UWB 채널 충돌)** 을 직접 측정하는 것이 목적이다.
2대만으로도 충돌 곡선의 기울기가 보이며, 그 기울기로 "몇 대에서 무너지는가"를 외삽한다.

---

## 1. 필요한 보드 (총 3대)

| 역할 | 펌웨어 | 설정 |
|---|---|---|
| 앵커 #1 (측정) | `anchor_collision_test.ino` | `TEST_ANCHOR_ID 1` |
| 앵커 #2 (간섭원) | `anchor_collision_test.ino` | `TEST_ANCHOR_ID 2` |
| 타깃 태그 | `Tag/tag_v1/tag_v1.ino` (그대로) | `MY_TAG_ID 1` |

> 태그 펌웨어는 **수정 불필요**. tag_v1은 어느 앵커 ID의 poll이든 수락하므로
> 앵커 1·2가 같은 태그(#1)를 동시에 노리면 충돌이 발생한다.

배치: 태그를 두 앵커에서 **같은 거리(예: 2~3m)** 에 고정. 세 보드 모두 시야선(LOS) 확보.
앵커 두 대는 서로 **RF 범위 안**(가까이, 예: 1~2m 간격)에 둬야 충돌이 일어난다.

---

## 2. 측정 절차

각 앵커는 USB로 PC에 연결하고 시리얼 모니터(115200)를 **각각** 연다.
출력의 `CSV,...` 줄을 파일로 저장하면 분석이 쉽다.

### 2-1. 기준선(Baseline) — 앵커 1대만

- 앵커 #2 **전원 OFF**. 앵커 #1만 동작.
- `POLL_DELAY_MS`를 아래 스윕 값으로 바꿔가며 각각 재플래시, 통계 줄 3~5개 기록.

### 2-2. 충돌(Contention) — 앵커 2대 동시

- 앵커 #1, #2 **둘 다 ON** (같은 `TEST_CHANNEL 5`, 둘 다 `TARGET_TAG_ID 1`).
- 같은 스윕을 반복. 앵커 #1의 시리얼에서 PER 기록.

### 2-3. 부하 스윕 값 (`POLL_DELAY_MS`)

```
150  → 50  → 20  → 10  → 5  → 2  → 0
```
값이 작을수록 에어타임을 더 먹어 충돌이 늘어난다.
`150`은 프로덕션 실제 값, `0`은 채널을 최대로 두드리는 극한.

---

## 3. 기록 양식

각 (앵커수 × delay) 조합마다 PER을 적는다:

| delay_ms | 1앵커 PER% | 2앵커 PER% | 차이(=충돌분) |
|---|---|---|---|
| 150 | | | |
| 50  | | | |
| 20  | | | |
| 10  | | | |
| 5   | | | |
| 2   | | | |
| 0   | | | |

- **1앵커 PER** = 순수 링크 품질/환경(NLOS·거리) 손실의 바닥값.
- **2앵커 PER − 1앵커 PER** = 순수 충돌로 인한 추가 손실. ← 이게 핵심 지표.

---

## 4. 결과 해석

| 관찰 | 의미 | 확장 함의 |
|---|---|---|
| 2앵커에서도 PER 거의 안 오름 | 채널이 아직 안 포화 | 단일 채널 여유 큼 — 공간재사용만으로 멀리 감 |
| delay 줄일수록 2앵커 PER 급상승 | 에어타임 포화 | 동시 활성 앵커 수에 명확한 상한 존재 |
| 150ms(실사용)에서 2앵커 PER이 유의미 | **현재 설정도 2대부터 충돌** | 같은 구역엔 채널분리(ch5/ch9)나 TDMA 필요 |

**확장 추정 공식 (근사):**
한 채널이 손실 없이 감당하는 초당 교환 수를 `C`라 하면,
1앵커 1태그의 초당 폴링 수(= 1000 / (교환시간 + delay))를 측정해 `C`를 역산하고,
`동시활성앵커 × 앵커당폴링율 < C` 를 만족하는 앵커 수가 한 채널·한 구역의 상한이다.
2채널(ch5/ch9)이면 상한 2배, 공간재사용으로 떨어진 구역은 독립 가산.

---

## 5. 다음 테스트로 연결

- **테스트 E (채널 분리):** 앵커 #2의 `TEST_CHANNEL`을 `9`로 바꿔 재측정.
  2앵커 PER이 1앵커 수준으로 떨어지면 → 채널 분리가 충돌을 없앤다는 증거.
  (주의: ch9는 최적 TX 파워가 다르다. 절대 거리보다 **PER 상대 비교**로만 해석.)
- **테스트 D (재사용 거리):** 두 앵커를 점점 멀리 떼며 2앵커 PER이 1앵커
  수준으로 수렴하는 거리 = 공간재사용 가능 거리.

---

## 6. 주의

- 이 측정 도구는 **프로덕션 펌웨어를 수정하지 않는다.** 별도 스케치다.
- DW3000 config·프레임·TWR 수식은 프로덕션과 동일 → 결과가 그대로 전이된다.
- WiFi/MQTT/릴레이는 제거됨(측정 순수성). 안전 로직은 들어있지 않으니
  **이 펌웨어로는 릴레이가 동작하지 않는다** — 순수 RF 측정 전용.
