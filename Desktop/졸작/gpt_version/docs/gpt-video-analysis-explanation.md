# GPT 기반 영상 AI 생성 판별 설명 자료

## 1. 영상을 넣으면 무슨 일이 일어나는가

1. 사용자가 웹 화면에서 영상 파일을 업로드합니다.
2. 서버는 영상을 저장하고, 영상 길이와 요청한 프레임 수를 기준으로 대표 프레임을 시간순으로 추출합니다.
3. 각 대표 프레임마다 원본 이미지와 프레임 간 변화량을 보여주는 heatmap 이미지를 만듭니다.
4. 서버는 영상 메타데이터와 프레임별 보조 수치를 계산합니다.
   - `temporalResidual`: 인접 프레임 사이의 전체 변화량
   - `edgeJump`: 엣지 구조 변화량
   - `sharpnessSwing`: 선명도 변화량
   - `colorShift`: 색 변화량
5. 서버는 GPT Responses API에 프롬프트, 원본 프레임 이미지, 선택적으로 heatmap 이미지를 함께 보냅니다.
6. GPT는 정해진 JSON 스키마에 맞춰 판정, 확률, 신뢰도, 근거, 반대 근거, 한계, 프레임별 설명을 반환합니다.
7. 화면은 결과를 전체 요약과 프레임별 근거 카드로 나누어 보여줍니다.

핵심은 영상을 통째로 한 번에 판별하는 것이 아니라, 대표 프레임과 보조 수치를 근거로 "확률적 판단"을 만드는 방식입니다.

## 2. 프롬프트는 어떻게 작성했는가

프롬프트는 `evidence-first`, 즉 근거 우선 방식으로 작성했습니다.

중요한 설계 원칙은 다음과 같습니다.

- 단정 금지: 법적 감정이나 최종 증거처럼 말하지 않고 확률적 판단으로 표현합니다.
- 대표 프레임 한계 명시: 전체 영상이 아니라 일부 프레임만 본다는 점을 GPT가 항상 고려하게 했습니다.
- 오판 방지: 보정, 압축, 저조도, 모션블러, 손떨림, 얕은 심도 같은 실제 촬영 요인을 AI 증거로 섣불리 판단하지 않게 했습니다.
- 증거-반증-한계 구조: AI 의심 근거뿐 아니라 실제 촬영처럼 보이는 반대 근거와 분석 한계를 함께 쓰게 했습니다.
- 점수 기준 고정: `ai_probability`가 모델 감각으로 흔들리지 않도록 0.00~1.00 구간별 의미를 지정했습니다.
- 스키마 강제: GPT가 자유문장으로 흩어지지 않게 JSON 스키마로 결과 형식을 고정했습니다.

프롬프트에 들어가는 입력은 크게 세 종류입니다.

- 영상 메타데이터: 파일명, 길이, 해상도, FPS, 코덱, 대표 프레임 수
- 프레임 이미지: 시간순으로 추출된 원본 대표 프레임
- 보조 수치: 프레임 간 변화량을 요약한 네 가지 수치

## 3. GPT는 무슨 근거로 판별하는가

GPT는 다음 관찰 항목을 중심으로 판단합니다.

- 구조 일관성: 얼굴, 손, 물체, 배경의 형태가 자연스럽게 이어지는지 봅니다.
- 텍스트와 패턴: 글자, 로고, 반복 무늬가 뭉개지거나 비현실적으로 반복되는지 확인합니다.
- 반사와 그림자: 빛 방향, 그림자, 반사 표현이 장면과 맞는지 봅니다.
- 질감과 디테일: 피부, 배경, 옷, 물체 표면이 지나치게 매끈하거나 균일한지 확인합니다.
- 카메라 특성: 노이즈, 압축 흔적, 렌즈 흐림, 조명 변화가 실제 촬영처럼 보이는지 봅니다.
- 프레임 간 안정성: 시간순 프레임에서 구조나 디테일이 갑자기 흔들리는지 확인합니다.
- 보조 수치: 변화량 수치가 시각적 단서와 맞는지 참고합니다.

단, 보조 수치는 단독 증거가 아닙니다. 사람이 움직이거나 카메라가 흔들려도 수치가 높아질 수 있으므로, GPT는 반드시 실제 프레임에서 보이는 시각 단서와 함께 해석하도록 되어 있습니다.

## 4. 결과를 사람들에게 설명하는 문장

"이 시스템은 영상을 업로드하면 시간순 대표 프레임을 뽑고, 각 프레임의 원본 이미지와 프레임 간 변화량을 GPT에 전달합니다. GPT는 프롬프트에 적힌 분석 기준에 따라 AI 의심 근거, 실제 촬영 쪽 반대 근거, 분석 한계를 분리해서 보고, 마지막에 AI 생성 가능성과 신뢰도를 확률로 제시합니다. 따라서 결과는 '무조건 AI다' 같은 단정이 아니라, 대표 프레임 기반의 근거 중심 리스크 평가입니다."

## 5. 발표 때 강조할 한계

- 전체 영상의 모든 프레임을 검사하는 방식이 아닙니다.
- 음성, 촬영 원본 메타데이터, 편집 이력까지 확인하는 법적 감정 도구는 아닙니다.
- 최신 AI 영상이나 강한 보정이 들어간 실제 영상은 구분이 어려울 수 있습니다.
- heatmap은 변화량 시각화이지 AI 생성의 직접 증거가 아닙니다.
- 최종 판단은 `AI_SUSPECTED`, `LIKELY_REAL`, `INCONCLUSIVE` 중 하나로 나오지만, 반드시 근거와 주의사항을 함께 봐야 합니다.
